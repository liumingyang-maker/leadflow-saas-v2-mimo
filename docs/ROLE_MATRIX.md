# Role Matrix

| 工作 | Codex | Reasonix/DeepSeek |
|---|---|---|
| 产品范围 | 负责 | 遵守 |
| 架构 | 负责 | 遵守 |
| 任务拆分 | 负责 | 接收 |
| 数据契约 | 审批 | 实现 |
| 功能代码 | 例外 | 负责 |
| 测试 | 规定/审查 | 实现 |
| 模板/CSS | 设计规格/审查 | 实现 |
| 安全审查 | 负责 | 修复 |
| UI 视觉审查 | 负责 | 修复 |
| Git/PR/合并 | 负责 | 禁止 |
| CI 故障 | 诊断 | 修复 |
| 发布决定 | 负责 | 禁止 |

## Worker packet 必含

任务 ID、目标、背景、允许/禁止文件、接口契约、验收标准、测试、命令、停止条件、证据格式。

## 修复纪律

Codex 审查失败后必须生成聚焦返工包。Reasonix 连续两轮无法修复时，Codex 只能做少于 30 行的集成修复；更大修改必须重新委派或上升为真正架构阻塞。
